What a responsive site is, and why responsiveness is important.

A responsive website is one that optimizes its layout to suit the size of the screen that the site is being view on. This is important because people are increasingly browsing the web on mobiles devices. A responsive website presents its content in a way that is best suited to the device you are using.

What mobile first design is, and why it's important.

Mobile first design is about structuring and designing your CSS so that the first thing to load is the layout for smaller screens, then using media query’s based on min-width’s the layout is reformed to suit wider screens.

This is important not only for consistent design between developers but also so that the devices that take the most load to process are mobile/smaller devices. This gives a faster response and load times on mobile devices that have less processing resource.

What frameworks are, and their pros and cons.

Frameworks are CSS template layout tools that contain some standard layouts blocks and can be built on to style your website in unique ways.

Frameworks save us time by taking care of basic layout requirements. But they can add unnecessary overhead to out page load and can complicate a simple design.

What a wireframe is and why we use it.

A wire frame is a design for a webpage broken up into layout blocks expressed by wireframe outlines. We use wireframes to as guide for laying out our webpage, it gives a guide for how to set out the building blocks of the page.

The aspects of your wireframes you found difficult to implement, and why

The hardest part of my wireframes to implement was the transition between desktop and mobile view. Specifically around transitioning the vertical column content to a horizontal layout. In the end I had to adjust my wire frame design to make implementing the design simpler.
